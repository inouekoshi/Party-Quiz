# 今後のマイルストーン (Future Milestones)

「Hobo Reunion Quiz」を同窓会本番で安全かつ盛大に成功させるための、今後の開発・運用マイルストーンです。

## フェーズ 1: コアロジックの完成 (完了)
* [x] Next.js と FastAPI の連携ボイラープレート作成
* [x] データベース (PostgreSQL + Prisma) のモデル設計とプロビジョニング
* [x] 参加者（ログイン、解答、ベッティング）画面の作成
* [x] プロジェクター（問題文、順位アニメーション表示）画面の作成
* [x] 管理者（出題、結果発表）コントロール画面の作成
* [x] WebSocket による全画面のミリ秒単位のリアルタイム同期
* [x] 速度ボーナス・マジョリティポイントの自動計算処理

---

## フェーズ 2: 耐障害性の強化とUXのポリッシュ (アルファ〜ベータ) 👈 **現在ココ**

### [ ] 1. 通信切断（リコネクト）への対応
会場のWifi環境等によっては参加者端末の通信が一時的に切れる可能性があるため、以下の対応を行う。
- WebSocketが切断された際の自動再接続処理の追加。
- 画面リロード時に、「現在どの問題が出題中か」という最新状態（GameState）をバックエンドから取得し直すエンドポイントの作成。

### [ ] 2. プロジェクターのリザルト演出拡張
- 単なるテキストの順序入れ替えだけでなく、ポイントがカウントアップ式で増えてから順位が入れ替わるようなリッチな演出を追加（余裕があれば）。
- 全12問等が完了した際の「最終表彰式画面（総合優勝、各特別賞）」の実装。

### [ ] 3. 管理者機能の強化
- 間違って採点してしまった場合の「特定の解答の取り消し・スコア修正機能」の作成。
- 回答時間を過ぎても解答していないチームがいた場合の「タイマー強制打ち切り」機能。

---

## フェーズ 3: 本番コンテンツの投入とインフラ配備 (本番準備)

### [ ] 1. サーバー環境へのデプロイ
現在ローカルで動いている環境を、外部からスマートフォン等でアクセスできるよう本番環境へ移行する。
- フロントエンド： Vercel へデプロイ
- バックエンド： Render, Railway 等へデプロイし、WebSocket対応のSSL（wss://）を設定。
- データベース： Render Managed DB や Supabase に本番用DBを構築。

### [ ] 2. 実際の問題（全12問想定）のマスターデータ作成
- 恩師のエピソードや、事前のアンケートをもとにした問題を Prisma の Seed สクリプトや SQL に書き起こし、本番環境のDBへ挿入。

### [ ] 3. 負荷テスト (Load Testing)
- 参加チームは8チーム（＝接続数はスマホ8台＋司会PC＋プロジェクターPCの概ね10端末）なのでトラフィックは小さく問題ない想定だが、同時に解答ボタンを押した際のミリ秒の速度判定に遅延が生じないか検証する。

---

## フェーズ 4: デハーサルと実践 (運用フェーズ)

### [ ] 1. オペレーションマニュアルの作成
- 当日、司会者（井上）がブラウザのタブをどう配置し、どのタイミングで画面を切り替えて「出題ボタン」を押すかという手順書の作成。
- BGM等の音響担当とのキュー出しタイミングの打ち合わせ。

### [ ] 2. デジタル・リザルト準備
- 大会終了後、各チームが自分たちの解答履歴（何を間違えてどのようにスコアが推移したか）を見返せるような「戦績ページ」を作成し、QRコードを発行しておく。

### [ ] 3. 会場リハーサル当日の接続テスト
- 実際の同窓会会場において、スクリーンへの投影サイズ、スマホからのアクセス速度、Wifiエリアの死角がないかをテストし、最終確認を行う。
